This guide outlines which branches **must NOT be deleted**, which require **instructor approval**, and which are **safe to delete after merge**.

## Team & Semester Branch Naming Convention

Team branches follow this structure:

```text
semesterYYYY-coursenumber-team#
```
or

```text
semesterYYYY-coursenumber-teamname
```

### Examples:
- `FALL2025-BA5200-Team1`
- `FALL2025-BA5200-Team3`
- `FALL2025-BA5200-Financial-Streamlining`
- `FALL2025-BA5200-RiskManagement`

### Notes:
- Used for **team or group assignments** within a semester.
- The ending may be a team number or a descriptive team name.
- When duplicates exist across semesters, the **most recent semester‚Äôs version** is the active one.
- These branches **must not be deleted without instructor approval**.

## Stale or Legacy Branches (Instructor Approval Required Before Deletion)

These branches are tied to **past semesters, earlier versions of the course, or old project cycles**.  
They may no longer be needed, but **must not be deleted without confirmation from the instructor**.

### Examples:
```text
SPRING2024-br-react-all-comps-2
SPRING2024-br-react-all-comps
MIS4000_Database
FALL2024-Scraping_API
Risk-Analysis-Spring-2025
```

## Main Branch

- The primary production branch.  
- All stable, complete work must eventually be merged here.

---

## General Rules

### 1. **Do NOT delete any branch unless:**
- The branch has been fully merged, **and**
- The author did not request it be kept, **and**
- It is not a semester/team branch, **and**
- It does not include the instructor‚Äôs name (`wall`), **and**
- It is not a legacy branch needing instructor approval.

### 2. **Always ask the instructor before deleting:**
- Any semester/team branch (e.g., `FALL2025-BA5200-Team3`)
- Any branch from older semesters (e.g., `SPRING2024-*`)
- Any branch with ‚Äúwall‚Äù in the name  
- Any unclear or historical branch

### 3. **Patch branches**
- Named like `username-patch-1`  
- Auto-generated by GitHub  
- Safe to delete after merge
